# B4_VENUE_MEMO — LLM vs B4 대결의 무대는 어디인가 (서술적 산술, 규칙 신설 0)

**판정: LLM-vs-B4 비교는 E2에서 성립하지 않는다 — no (신규 스냅샷 112개 중 0개,
실험군 48개 중 0개가 B4 점수 계산 가능).**

> Authored by Claude Code, 2026-07-13 (D60, 무호출). 이 메모는 동결 레지스트리
> (candidates.json)와 로컬 제출 캐시의 날짜 산술만이다 — E2 설계·스펙·판정
> 규칙을 만들지도 바꾸지도 않는다.

## 1. 산술 입력 (전부 기존 동결값)

- **E2 스냅샷 그리드** (docs/EARLINESS_DESIGN.md §1, 동결): 스냅샷 j의 컷오프 =
  케이스 컷오프 이전 j번째 최신 10-K/10-Q filed일 + 1일; 케이스당 상한 8
  (OFIX 2·LOGI 6), 실험군 7 (MON 제외) + RP-01 대조군 8.
- **B4 점수 계산 가능 하한** (specs/B4_short_interest.md §6, 사전 등록 산술):
  FINRA 데이터 하한 결제일 2017-12-29 + trailing 12개월 + LAG 14일 ⇒ 컷오프
  **≥ 2019-01-27**. (약한 하한 — 보고서 1건이라도 피처 진입: ≥ 2018-01-12.)

## 3. 귀결 (기존 규칙의 적용 서술 — 신설 없음)

1. **E2 판정에서 §4b는 스스로 비성립을 기록한다**: engine_verdict가
   `b4_comparison.valid=false` + 사유(커버리지)를 남기고 기본 판정(§4
   a/b/c)은 B4 무관하게 진행된다. `b4_dominated` 강등은 E2 산출물에서
   **트리거될 수 없다** — 이것은 결함이 아니라 사전 등록 산술의 정직한
   귀결이다.
2. **대결의 유일한 무대 = 전향 seal** (screener
   specs/SEALED_FORECAST_PROTOCOL.md §5 metric 3 + §5b):
   - 첫 seal 2026-11-15 (B4 top-30 동봉 의무) → 첫 판정 증거(4분기 창 만료)
     **≈ 2027-11**;
   - 4연속 분기 stage-gate의 최초 개방 가능 시점: 2026-11·2027-02·2027-05·
     2027-08 seal의 창이 각각 만료되는 **≈ 2028-08**.
3. **결합 조항의 관할**: B4 스펙 §7(D55, 완화 금지)의 "비교 성립 시 LLM≤B4
   → 엔진 결정 입력"은 seal 판정과 미래의 커버리지 있는 tier에서 발효한다.
   E2는 그 무대가 아니며, E2 발사 결정(예산 112~160호출)의 기대값 계산에서
   "무료 신호 대결 해소"를 편익으로 계상하면 안 된다 — E2가 사는 것은
   LLM vs **B3** 리드타임(§4 기본 판정)이지 B4가 아니다.
