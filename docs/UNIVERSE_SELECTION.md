# UNIVERSE_SELECTION.md — EQ 모니터 유니버스 기계적 선정 프로토콜 (사전 등록)

> Authored by Claude Code, pending human sign-off. 2026-07-16.
> **이 문서는 어떤 후보 기업도 조회하기 전에 커밋된다 — 커밋 타임스탬프가
> 사후 선택(hindsight selection) 방어의 전부다.** GIL 메모의 교훈: 선정이
> 기계적 기준보다 먼저 오면, 그 산출물은 영원히 "사후 선택" 각주를 달고 다닌다.
> 이 문서 커밋 이전에 생성된 후보 목록은 존재하지 않으며, 존재해서는 안 된다.
> 기준 확정(서명) 전 후보 열거 금지.

## 0. 지위와 발효

- 대상: 향후 EQ 모니터 정기 발행의 스크리닝 유니버스 (PROJECT.md §3 — 8~15개).
- 발효: **소유자 서명 시** (아래 §6 선택지 확정 포함). 서명 전에는 어떤 세션도
  EDGAR SIC browse·후보 조회를 실행하지 않는다.
- 개정: 첫 열거(enumeration) 실행 이후에는 **FREEZE_REV 문서 + D-엔트리로만**
  (PROJECT.md §5-6 이력 공개). 열거 전 수정은 일반 커밋으로 가능하되 서명 재요청.

## 1. 모집단 정의 (기계적 — 재량 지점 0)

열거 시점 T₀(서명 후 첫 실행일)에 다음을 **전부** 만족하는 SEC 등록사:

1. **폼 요건**: 최근 24개월 내 10-K ≥ 1건 **및** 10-Q ≥ 2건 제출 (EDGAR
   submissions API 기준). — 40-F/20-F 외국사 제외 사유: B3 메타신호(8-K 빈도·
   NT·정정)와 분기 XBRL 그리드가 10-K/10-Q 체계를 전제한다. 6-K 체계는 비교
   불능(GIL 메모에서 실측).
2. **섹터**: §6에서 확정하는 SIC 코드 집합에 속함 (EDGAR browse-edgar SIC 기준).
3. **XBRL 이력**: companyfacts에 XBRL 재무 사실이 있는 10-K/10-Q ≥ **8건**,
   그중 10-K ≥ **2건**. (근거: EARLINESS_PLAN §1 최소 바닥 "잔존 제출 ≥6·10-K ≥1"
   보다 한 단계 위 — 라이브 모니터는 궤적 채점 여력이 필요.)
4. **규모**: 최근 `dei:EntityPublicFloat` ≥ **$1B** (XBRL 기계 판독 — 시가총액
   대신 float를 쓰는 이유: companyfacts 안에서 폐쇄적으로 계산 가능, 외부
   시세 소스 불요).

## 2. 제외 규칙 (기계적)

1. **오염 격리 — 별도 트랙 강등**: T₀ 기준 트레일링 24개월 내 다음 중 하나라도
   있으면 본 유니버스에서 제외하고 `post-hoc selection` 트랙(§4)으로만 다룰 수
   있다: (a) 8-K Item 4.02 제출, (b) 공개 공매도 리포트 대상(EDGAR 밖 사건이므로
   기계 검출 불가 — **알게 된 순간** 소급 강등, GIL 판례), (c) SEC AAER/집행 대상.
2. **자기 오염**: Cycle-1 케이스 세트(실험군·대조군·홀드아웃·E1 대조군)에 이미
   등장한 회사 — 이 저장소가 해당 회사에 대한 판단을 이미 공개했으므로 제외
   (FUTURE_CYCLE_PROTOCOL §0의 소각 원리).
3. 이력 미달(신규 상장 등)은 §1-3에서 자동 탈락 — 수기 판단 없음.

## 3. 순위·타이브레이커 (재량 제거)

- 적격 초과 시(>15): SIC 버킷별로 **float 내림차순** 정렬 후 버킷 라운드로빈으로
  **12개** 선정 (8~15의 중앙 고정 — 소유자 서명 시 변경 가능). 나머지는 순서
  보존 대기 목록(alternates)으로 기록 — 탈락·교체 시 승격 순서가 이미 고정.
- 동률: CIK 오름차순 (결정론).
- 선정 결과가 나온 뒤 목록을 보고 개별 기업을 넣고 빼는 행위 금지 — 빼야 할
  사유(오염 발견 등)가 생기면 §2 규칙 인용 + 기록 후 대기 목록 자동 승격.

## 4. post-hoc selection 트랙 (GIL 형)

- §2-1로 강등된 회사도 분석은 가능하되, 산출물에 **"선정은 사건 인지 후 —
  무작위 스크리닝 적중 아님"** 명시 + 입력 데이터 컷오프를 사건 이전으로 코드
  강제(look-ahead 가드)해야 한다. GIL 메모 v1이 이 트랙의 규격 판례다.
- 이 트랙의 산출물은 유니버스 정기 모니터 실적(Tier 2 지표)으로 집계하지 않는다.

## 5. 열거 실행 규약 (서명 후)

- 원시 응답 보존: SIC browse·submissions 응답 원문을 `data/candidates/universe/`
  아래 저장 + provenance(질의 URL·시각) 기록 — rp08 대조군 풀 수집 패턴 승계.
- 네트워크는 소유자 입회 승인 하에만 (§5-1 판례 Q-E03 — 무인 fetch의 조용한
  look-ahead 위험). 데이터 로딩은 `pipeline/cutoff_guard.py` 경유.
- 생존 편향 기록: 현재 시점 열거는 상장폐지사를 구조적으로 누락한다(Q-F08).
  라이브 모니터는 "지금 상장된 회사"가 정의상 대상이므로 결함이 아니지만,
  백테스트 대조군과의 비교 서술에는 이 각주를 동반한다.

## 6. 소유자 확정 선택지 (서명 시 1개 확정 — 확정 전 후보 조회 금지)

**섹터 SIC 집합** — "AI 인프라 인접" (PROJECT.md §3)의 기계적 번역 후보:

- **(A) 기본 — 하드웨어·전력 사슬 협의(狹義)**: 3674(반도체)·3672(PCB)·
  3661/3663/3669(통신장비)·3571(컴퓨터)·3572(스토리지)·3576(네트워크 장비)·
  3577(주변장치)·3612/3613(전력 배전·개폐)·3621(발전기·모터)·3585(냉각)·
  4911(전력 유틸리티).
  근거: 물리적 AI 인프라(칩→네트워크→전력→냉각) 전 사슬, 회계적으로 재고·
  장기계약·capex가 무거워 EQ 신호 밀도가 높은 업종.
- **(B) 소프트웨어 포함 광의**: (A) + 7372(패키지 SW)·7371/7379(IT 서비스).
  근거: 커버리지 확대. 반론: 무형·구독 회계는 체크리스트(CL1~CL8)의 재고·
  마진 축이 자주 insufficient_data로 빠진다 (backtest 실측 — CL4 대조군
  insufficient 17/45).
- **(C) (A) − 유틸리티**: 4911 제외. 근거: 규제 회계(regulatory assets)는
  일반 제조업 체크리스트와 이질적.

기본값(무응답 시): **(A)**. 단 §0에 따라 무응답 상태로는 열거를 시작하지 않는다
— 이 문서는 서명이 있어야 발효한다.
