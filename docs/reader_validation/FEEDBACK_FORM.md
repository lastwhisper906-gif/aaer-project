# Reader feedback form (Tier 3 — value validation)

> 7 questions max. Q1–Q3 are fixed for every reader (they are the project's
> pre-registered value criteria, PROJECT.md §3/§9); pick the remaining 1–2 by
> reader type. Answers can be one line each — 5 minutes total.

## Core (all readers — fixed)

1. **Overclaim check.** Does any part of the conclusion read as an overclaim
   to you? If yes, quote the sentence.
2. **First attack point.** Where would you attack the methodology first?
   (One sentence — e.g., sample construction, grading, leakage, statistics.)
3. **Who pays?** Who, concretely, would spend money or time because of this
   result — and on what? ("Nobody" is a useful answer; then: what is missing
   for that to change?)

## By reader type (pick 1–2)

### Accounting / audit practice

4. Which of the flagged signals (e.g., the Hub Group unrecorded-payables
   pattern) would have been visible to you earlier through normal procedures —
   and which would not?
5. Is the G2 "restatement / non-reliance, never confirmed fraud" framing for
   living companies adequate, or still too suggestive?

### ML / LLM evals

4. Is the freeze-then-run + pre-committed conclusion-rule design sufficient to
   trust the R3/R4/H2 readouts, or is there a contamination channel we have
   not bounded?
5. Given the instrument-bias table (name-ID vs outcome-knowledge vs
   perturbation), what instrument would you add before believing "capability,
   not memorization"?

### Investment / research

4. At a real-world base rate (~0.7% fraud prevalence), the backtest AUC does
   not translate into screening PPV. Would a forward watchlist with
   timestamped predictions change your view — or is N too small to matter?
5. What would this signal have to show, and at what precision, before it
   changes an actual allocation or diligence decision you make?
