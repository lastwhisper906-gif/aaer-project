# specs/cross_grader.md — 교차 패밀리 채점자 스팟체크 사전 등록 (WS-7 / F-8, SPEC ONLY)

> **launch-ready 동결 — 이 세션에서 실행하지 않는다.** 발사는 소유자 게이트
> **Q-F07** (채점자 모델 선정 + API 키 + 예산 ~20호출) 서명 후, **이미
> launch-ready인 E4 교차모델 런과 같은 배치에서** — E4를 중복하지 않는다
> (E4 = 피평가자 교차, 본 스펙 = 채점자 교차; 상호 보완 별개 축).

## 1. 위협 모델

피평가자(claude-sonnet-5)와 채점자(fable-5)가 **같은 모델 패밀리**다 —
주관 차원 채점이 패밀리 공유 편향(같은 서술 스타일을 근거 충실로 읽는
경향 등)에 오염될 수 있다. dim1(확률 밴드)은 기계적 밴딩이라 이 위험이
없고, **dim2(mechanism-match)·dim4(evidence-groundedness)** 두 주관
차원만 노출된다.

## 2. 표본 설계 (사전 등록)

- **n = 20**. n=10 기각 근거 (사전 등록): κ의 신뢰구간이 n=10에서는 어떤
  실질 결론도 경계할 수 없을 만큼 넓다 — n=20이 "κ ≥ 0.6 유지/미달"의
  단측 판정에 필요한 최소 실용 규모.
- **층화 규칙**: 모집단 = human_finalized 채점 레코드 전체
  (`scoring/grades*/`). 층 = tier (wave-1 / wave-2 / holdout+E1 대조군) ×
  주관 차원 등급 수준 (dim2 {0,1,2} · dim4 {하/중/상 밴드}) — 비례 배분,
  단 각 tier 최소 4건 보장. 셀 내 선정은 결정론 seed **20260712**로
  case_id 정렬 후 균등 간격 추출 (자의 선정 봉쇄).
- 채점 프레임: 동결 루브릭 앵커 프롬프트 **원문 그대로** (재해석 금지),
  **피평가자 모델 정체 비공개** (채점자에게 어느 모델의 출력인지 알리지
  않는다).

## 3. 채점자 (소유자 게이트)

- **실행 시점의 최강 가용 비-Anthropic 모델** — 소유자가 Q-F07에서 선정
  (API 키·과금 = 소유자 인프라, 세션 권한 밖). 스펙은 모델을 지금 지명하지
  않는다 (실행 시점의 시장 상태에 종속 — 사전 지명이 오히려 자의적).
- 호출 산술: 20 샘플 × 1 채점 = **~20 호출** (차원 2개는 단일 호출 내
  동시 채점 — 기존 grader_runner 프롬프트 구조 유지).

## 4. 지표와 사전 등록 판독 (계산 전 고정)

- **Cohen's κ, 차원별** (dim2: 3수준 가중 없음 unweighted · dim4: 밴드
  수준 unweighted — 동률 처리 포함 표준 정의).
- **κ ≥ 0.6 (양 차원 모두)** → 현 결론 유지 — "채점자 교차 일치" 각주 병기.
- **κ < 0.6 (어느 한 차원)** → 해당 주관 차원 결과를 "grader-dependent"로
  강등하는 **diff 제안 의무** (diff-only, review_packets/ + OWNER_QUEUE —
  직접 수정 금지). 발행 동결값 자체는 불변 (강등은 해석 층).
- 중간 판정 없음 — 경계 0.6 단일 (Landis-Koch 'substantial' 하한).

## 5. 실행 규율 (발사 시)

- freeze-commit-then-run: 표본 20건의 case_id 목록을 **채점자 호출 전에
  커밋** (층화 규칙의 기계 적용 결과 — 사후 교체 봉쇄).
- 결과는 `runs/` 신규 하위 디렉토리 + 전역 MANIFEST 재생성 + κ 계산
  스크립트는 결정론 Python.
- E4와 같은 배치 실행 시에도 산출물·D-엔트리는 분리 기록.
